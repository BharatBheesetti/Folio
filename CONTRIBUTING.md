# Contributing to Folio

Thanks for your interest in contributing.

## Getting started

```bash
git clone https://github.com/BharatBheesetti/Folio.git
cd Folio
npm install
npm start
```

## Guidelines

- Keep it minimal. Folio is intentionally small — resist adding features that don't serve the core "read markdown beautifully" mission.
- Test your changes by opening a variety of markdown files (code blocks, tables, nested lists, long documents).
- Follow the existing code style. No linter is configured — just match what's there.

## Reporting bugs

Open an [issue](https://github.com/BharatBheesetti/Folio/issues) with:
- What you expected vs. what happened
- Steps to reproduce
- Windows version and Folio version

## Pull requests

1. Fork the repo and create a branch from `main`.
2. Make your changes.
3. Test with `npm start`.
4. Open a PR with a clear description of the change.
